# Phase 0 Plan — Foundations: schema + store

## Files
- `api/models.py` — 5 SQLAlchemy tables (LLD §1): `graphs`, `graph_versions`, `runs`, `node_runs`, `node_logs`. SQLite types: `String` hex ids, `JSON`, `DateTime`. Imports `Base` from `config/db.py`.
- `api/engine/__init__.py`, `api/engine/store.py` — `Store` wrapping `Database`. Methods: `get_run`, `get_nodes`, `get_node`, `create_run`, `create_node`, `set_status`, `set_output`, `set_input`, `log`. Only SQL touchpoint.
- `api/alembic/versions/<rev>_dag_schema.py` — autogenerated migration for the 5 tables (on top of empty init).
- `api/tests/__init__.py`, `api/tests/conftest.py` (in-memory/temp sqlite fixture), `api/tests/test_store.py`.

## Design
- `set_status` enforces LLD §2 allowed transitions via `ALLOWED: set[(from,to)]`; raises `ValueError` on illegal (e.g. `done→running`).
- Per-op session: each method opens `db.session()` (autocommit). Matches persist-before-return. ponytail: batch later if perf matters.
- ids = `uuid4().hex`; timestamps = `datetime.now(utc)`.

## Test (`test_store.py`)
1. Insert graph+version+run+node_run → read back via `get_run`/`get_nodes`.
2. `set_status` legal (`pending→running`) ok.
3. `set_status` illegal (`done→running`) raises.

## Gate
`alembic upgrade head` clean (5 tables) + `pytest api/tests/test_store.py` green.
